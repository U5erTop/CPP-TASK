# Стандартная библиотека шаблонов (STL) в C++

---

## 1. Введение в архитектуру STL

**STL (Standard Template Library)** — это мощная библиотека обобщенных классов и функций, являющаяся ядром стандарта C++.
В основе STL лежат три главных компонента (иногда выделяют больше):

1. **Контейнеры (Containers)** — классы для хранения данных (`vector`, `list`, `map` и др.).
2. **Алгоритмы (Algorithms)** — функции для обработки данных в контейнерах (поиск, сортировка, копирование).
3. **Итераторы (Iterators)** — объекты, работающие как указатели. Они связывают алгоритмы и контейнеры, позволяя алгоритмам быть независимыми от внутреннего устройства контейнера.

Дополнительные компоненты: **Адаптеры**, **Функторы (Функциональные объекты)** и **Аллокаторы**.

---

## 2. Итераторы: Связующее звено

Итератор — это абстракция указателя. Контейнеры предоставляют методы `begin()` (указывает на первый элемент) и `end()` (указывает на элемент **за** последним).

### Категории итераторов:

1. **Input (Ввода)** — только чтение, движение только вперед (`++`).
2. **Output (Вывода)** — только запись, движение только вперед.
3. **Forward (Прямого доступа)** — чтение/запись, движение вперед (`++`).
4. **Bidirectional (Двунаправленные)** — чтение/запись, движение вперед и назад (`++`, `--`). Поддерживаются `list`, `set`, `map`.
5. **Random Access (Произвольного доступа)** — все вышеперечисленное + арифметика указателей (`+`, `-`, `[]`, `<`, `>`). Поддерживаются `vector`, `deque`, `array`.

```cpp
#include <vector>
#include <iostream>

int main() {
    std::vector<int> v = {10, 20, 30};

    // Классический проход с итератором
    for (std::vector<int>::iterator it = v.begin(); it != v.end(); ++it) {
        std::cout << *it << " ";
    }

    // C++11: range-based for (под капотом использует те же итераторы)
    for (int x : v) std::cout << x << " ";
}
```

## 3. Последовательные контейнеры (Sequence Containers)

Они хранят элементы в строгом линейном порядке.

### `std::vector` (Динамический массив)

- **Плюсы:** Быстрый доступ по индексу `O(1)`. Быстрое добавление в конец.
- **Минусы:** Медленная вставка/удаление в середину или начало `O(N)`. При переполнении происходит переаллокация памяти.
- _Заголовок:_ `<vector>`

### `std::list` (Двусвязный список)

- **Плюсы:** Быстрая вставка/удаление в ЛЮБОМ месте `O(1)` (если есть итератор).
- **Минусы:** Нет произвольного доступа (нельзя сделать `list[5]`). Медленный поиск.
- _Заголовок:_ `<list>`

### `std::deque` (Двусторонняя очередь)

- **Плюсы:** Быстрая вставка/удаление и в конец, и в НАЧАЛО `O(1)`. Поддерживает доступ по индексу. Размещается в памяти кусками (chunks).
- _Заголовок:_ `<deque>`

_(Также существуют `std::array` для массивов фиксированного размера и `std::forward_list` для односвязных списков)._

---

## 4. Ассоциативные контейнеры (Associative Containers)

Основаны на бинарных деревьях поиска (обычно красно-черных). Данные хранятся в **отсортированном** виде. Поиск, вставка и удаление работают за `O(log N)`.

### `std::set` (Множество)

- Хранит только уникальные значения.
- _Заголовок:_ `<set>`

### `std::map` (Словарь / Ассоциативный массив)

- Хранит пары "Ключ-Значение". Ключи уникальны.
- Перегружен оператор `[]`: `m["apple"] = 5;` (если ключа нет, он будет создан).
- _Заголовок:_ `<map>`

_(Существуют также `multiset` и `multimap`, разрешающие дубликаты ключей)._

```cpp
#include <map>
#include <string>

int main() {
    std::map<std::string, int> ages;
    ages["Alice"] = 25;
    ages["Bob"] = 30;

    for (const auto& pair : ages) {
        // pair.first - ключ (const string), pair.second - значение (int)
        std::cout << pair.first << ": " << pair.second << "\n";
    }
}
```

---

## 5. Неупорядоченные ассоциативные контейнеры (C++11)

Основаны на **хэш-таблицах**. Элементы не отсортированы, но вставка, поиск и удаление в среднем работают за **О(1)** (константное время).

- `std::unordered_set`
- `std::unordered_map`
- _Заголовки:_ `<unordered_set>`, `<unordered_map>`

_Правило:_ Если вам не важен порядок обхода элементов (от меньшего к большему), всегда предпочитайте `unordered_map` обычному `map` ради скорости.

---

## 6. Адаптеры контейнеров (Container Adaptors)

Это не самостоятельные контейнеры, а "обертки" над последовательными контейнерами, предоставляющие специфический интерфейс. Они **не поддерживают итераторы**!

1. **`std::stack` (Стек):** LIFO (Last In, First Out). Методы: `push`, `pop`, `top`. (Обертка над `deque` или `vector`).
2. **`std::queue` (Очередь):** FIFO (First In, First Out). Методы: `push`, `pop`, `front`, `back`. (Обертка над `deque` или `list`).
3. **`std::priority_queue` (Очередь с приоритетом):** Выдает максимальный элемент (основана на структуре данных Heap). (Обертка над `vector`).

---

## 7. Алгоритмы STL

Большинство алгоритмов находятся в заголовке `<algorithm>`. Они принимают итераторы (начало и конец диапазона). Конец диапазона всегда "не включительно" `[begin, end)`.

```cpp
#include <vector>
#include <algorithm>

int main() {
    std::vector<int> v = {4, 1, 3, 5, 2};

    // Сортировка по возрастанию
    std::sort(v.begin(), v.end());

    // Поиск элемента
    auto it = std::find(v.begin(), v.end(), 3);
    if (it != v.end()) std::cout << "Нашли 3!\n";

    // Подсчет количества элементов, равных 5
    int c = std::count(v.begin(), v.end(), 5);
}
```

### Популярные алгоритмы:

- Немодифицирующие: `find`, `count`, `for_each`, `all_of`, `any_of`.
- Модифицирующие: `copy`, `transform`, `replace`, `remove`, `fill`.
- Сортировки и кучи: `sort`, `stable_sort`, `binary_search`, `make_heap`.

---

## 8. Функторы и Предикаты

Что если мы хотим отсортировать вектор по убыванию? Алгоритм `std::sort` принимает третьим аргументом **функтор** или функцию сравнения.

**Функтор** — это объект класса, в котором перегружен `operator()`.
**Предикат** — это функтор или функция, возвращающая `bool`. Унитарный предикат принимает 1 аргумент (для `std::find_if`), бинарный — 2 аргумента (для `std::sort`).

```cpp
#include <vector>
#include <algorithm>
#include <functional> // Здесь лежат встроенные функторы типа std::greater<T>

// Свой функтор
struct Descending {
    bool operator()(int a, int b) const {
        return a > b;
    }
};

int main() {
    std::vector<int> v = {1, 5, 2};

    // Сортировка своим функтором
    std::sort(v.begin(), v.end(), Descending());

    // Сортировка встроенным функтором STL
    std::sort(v.begin(), v.end(), std::greater<int>());
}
```

## 9. Лямбда-выражения (C++11)

Писать отдельные классы-функторы для простых проверок утомительно. C++11 ввел анонимные функции — **лямбды**.

Синтаксис: `[захват](параметры) -> возвращаемый_тип { тело }`

```cpp
std::vector<int> v = {1, 2, 3, 4, 5};

// Считаем четные числа с помощью лямбды
int evens = std::count_if(v.begin(), v.end(), [](int x) {
    return x % 2 == 0;
});

// Захват переменных из внешнего контекста (по ссылке)
int sum = 0;
std::for_each(v.begin(), v.end(), [&sum](int x) {
    sum += x;
});
```

## 10. Строки (std::string)

Хоть строки и выделяют отдельно, `std::string` — это по сути `std::basic_string<char>`, который работает как `std::vector<char>`. Он поддерживает итераторы, к нему применимы алгоритмы STL (например, `std::reverse(s.begin(), s.end())`).

## Рекомендации по выбору контейнера

1. **По умолчанию выбирайте `std::vector`** (лучшая локальность данных, работает быстрее всего за счет кэша процессора).
2. Если нужны частые вставки в начало и конец — `std::deque`.
3. Если нужна сортировка "на лету" и поиск по ключу — `std::map` или `std::set`.
4. Если важен только быстрый поиск по ключу O(1) и порядок не важен — `std::unordered_map`.
5. `std::list` нужен _только_ тогда, когда происходит огромное количество вставок в СЕРЕДИНУ гигантских списков (при условии, что у вас УЖЕ есть итератор на это место), или когда требуется, чтобы ссылки/итераторы на элементы не инвалидировались никогда.
