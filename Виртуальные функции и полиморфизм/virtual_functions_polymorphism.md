# Виртуальные функции и полиморфизм в C++

---

## 1. Введение в полиморфизм

Слово **полиморфизм** означает "множество форм". В C++ он поддерживается в двух видах:

1. **Во время компиляции (раннее связывание):** перегрузка функций, перегрузка операторов и шаблоны.
2. **Во время выполнения (позднее связывание):** виртуальные функции.

Эта лекция посвящена второму типу — **динамическому полиморфизму**, который позволяет использовать один и тот же интерфейс для работы с объектами различных типов в иерархии наследования.

---

## 2. Указатели на базовый класс

Фундаментальное правило C++, на котором строится динамический полиморфизм: **указатель на базовый класс может указывать на объект любого производного от него класса**.

```cpp
class Base {
public:
    void show() { std::cout << "Базовый класс\n"; }
};

class Derived : public Base {
public:
    void show() { std::cout << "Производный класс\n"; }
};

int main() {
    Base b;
    Derived d;

    Base* ptr;
    ptr = &b;
    ptr->show(); // Выводит: Базовый класс

    ptr = &d;    // Указатель Base* указывает на объект Derived
    ptr->show(); // Выводит: Базовый класс (Раннее связывание!)
}
```

**Почему выводится "Базовый класс"?**
Компилятор использует _раннее связывание_ (статическое). Он видит, что `ptr` имеет тип `Base*`, и жестко прописывает вызов `Base::show()`. Чтобы тип определялся не по типу указателя, а по **типу самого объекта** в памяти, нужны виртуальные функции.

---

## 3. Виртуальные функции и позднее связывание

Чтобы включить позднее связывание, метод в базовом классе должен быть объявлен с ключевым словом `virtual`.

```cpp
class Base {
public:
    virtual void show() { std::cout << "Базовый класс\n"; }
};

class Derived : public Base {
public:
    void show() override { std::cout << "Производный класс\n"; }
};

int main() {
    Derived d;
    Base* ptr = &d;
    ptr->show(); // Выводит: Производный класс (Позднее связывание!)
}
```

**Особенности:**

- `virtual` пишется только в объявлении функции внутри класса (в `.h` файле, если реализация в `.cpp`).
- Если функция объявлена виртуальной в базе, она остается виртуальной во **всех** наследниках на любой глубине (но хорошим тоном считается писать `override`).

---

## 4. Как работает полиморфизм под капотом (vtable и vptr)

Когда класс содержит хотя бы одну виртуальную функцию, компилятор создает:

1. **vtable (Таблицу виртуальных функций):** статический массив указателей на реализации виртуальных функций для этого класса.
2. **vptr (Скрытый указатель):** добавляется в каждый объект этого класса. Указывает на `vtable`.

**Процесс вызова (ptr->show()):**

1. Программа идет по указателю `ptr` к объекту.
2. Читает скрытый указатель `vptr`.
3. Переходит по `vptr` в таблицу `vtable` объекта `Derived`.
4. Находит там адрес `Derived::show()` и вызывает его.

_Именно поэтому полиморфизм работает только через указатели и ссылки. При передаче объекта по значению происходит "срезка" (object slicing), vptr меняется на базовый, и полиморфизм теряется._

---

## 5. Спецификаторы override и final (C++11)

До стандарта C++11 легко было допустить ошибку при переопределении функции (например, забыть `const` или изменить тип аргумента). В итоге компилятор создавал новую функцию (сокрытие), а полиморфизм ломался.

### Спецификатор override

Заставляет компилятор строго проверить, действительно ли метод переопределяет виртуальную функцию из базового класса.

```cpp
class Base {
public:
    virtual void func(int x) const {}
};

class Derived : public Base {
public:
    // void func(int x) override {} // ОШИБКА: забыт const
    void func(int x) const override {} // OK
};
```

### Спецификатор final

Запрещает дальнейшее переопределение функции в классах-наследниках. Может применяться и ко всему классу (запрещает наследование).

```cpp
class Base {
public:
    virtual void step() final {} // Никто больше не сможет переопределить step()
};

class Leaf final : public Base { // Никто не сможет наследоваться от Leaf
};
```

## 6. Чисто виртуальные функции и абстрактные классы

Часто в базовом классе нет смысла реализовывать функцию (например, как нарисовать абстрактную "Фигуру"?). Такую функцию делают **чисто виртуальной**, приравняв к нулю `= 0`.

```cpp
class Shape {
public:
    virtual void draw() const = 0; // Чисто виртуальная функция
};
```

Класс, содержащий хотя бы одну чисто виртуальную функцию, называется **абстрактным**.

- Нельзя создать объект абстрактного класса: `Shape s; // Ошибка`.
- Можно создавать указатели/ссылки: `Shape* p; // OK`.
- Любой наследник **обязан** реализовать все чисто виртуальные функции, иначе он тоже станет абстрактным.

---

## 7. Виртуальные деструкторы (Критическое правило)

**Правило Шилдта:** Если в классе есть хотя бы одна виртуальная функция (или он предполагается как базовый полиморфный), его деструктор **должен быть виртуальным**.

Если удалить объект производного класса через указатель на базовый класс без виртуального деструктора, произойдет **утечка памяти** (вызовется только деструктор базового класса).

```cpp
class Base {
public:
    // Без virtual здесь будет вызвана только очистка Base!
    virtual ~Base() { std::cout << "Base уничтожен\n"; }
};

class Derived : public Base {
    int* data;
public:
    Derived() { data = new int[100]; }
    ~Derived() override {
        delete[] data;
        std::cout << "Derived уничтожен\n";
    }
};

int main() {
    Base* ptr = new Derived();
    delete ptr; // Корректно: сначала ~Derived(), затем ~Base()
}
```

_Конструкторы, в отличие от деструкторов, виртуальными быть **не могут**._

---

## 8. Ковариантные типы возвращаемого значения

При переопределении виртуальной функции тип возвращаемого значения должен совпадать с базовым. **Исключение:** переопределенная функция может возвращать указатель/ссылку на класс, производный от того, который возвращала базовая функция.

Это часто используется для паттерна "Клонирование" (виртуальный конструктор):

```cpp
class Base {
public:
    virtual Base* clone() const { return new Base(*this); }
};

class Derived : public Base {
public:
    // Возвращаем Derived*, а не Base* - это ковариантность!
    Derived* clone() const override { return new Derived(*this); }
};
```

---

## 9. Опасности: виртуальные функции в конструкторе

**Никогда не вызывайте виртуальные функции внутри конструкторов и деструкторов!**

В момент работы конструктора базового класса объект производного класса **ещё не существует** (его поля не инициализированы). Если из `Base::Base()` вызвать виртуальную функцию, сработает версия `Base`, а не `Derived`, что часто является неожиданностью для программиста.

```cpp
class Base {
public:
    Base() { init(); } // Вызовется Base::init() !
    virtual void init() { std::cout << "Base init\n"; }
};

class Derived : public Base {
public:
    void init() override { std::cout << "Derived init\n"; }
};

int main() {
    Derived d; // Выведет "Base init"
}
```
