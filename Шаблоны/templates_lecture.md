# Шаблоны в C++ (Templates)

---

## 1. Введение в шаблоны (Обобщенное программирование)

Допустим, вам нужно написать функцию, меняющую местами два значения (`swap`). Без шаблонов пришлось бы писать отдельные перегрузки для `int`, `double`, `char` и всех пользовательских классов.
**Шаблоны (Templates)** решают эту проблему. Они позволяют создавать "каркас" (макет) функции или класса, в котором типы данных оставляются неопределенными до момента их использования. Компилятор сам сгенерирует нужные версии кода (инстанцирует их) на основе переданных типов.

Это основа **обобщенного программирования (Generic Programming)**, на котором построена вся Стандартная библиотека шаблонов (STL).

---

## 2. Шаблонные функции

Синтаксис объявления шаблонной функции:

```cpp
template <typename T> // или template <class T>
возвращаемый_тип имя_функции(параметры) {
    // тело функции
}
```

_Примечание: в контексте параметров шаблона слова `typename` и `class` абсолютно эквивалентны. Шилдт часто использовал `class`, но современный стандарт рекомендует `typename`._

### Пример: функция swap

```cpp
template <typename T>
void mySwap(T& a, T& b) {
    T temp = a;
    a = b;
    b = temp;
}

int main() {
    int x = 10, y = 20;
    mySwap(x, y); // Компилятор неявно выводит T = int и создает void mySwap(int&, int&)

    double d1 = 1.5, d2 = 3.14;
    mySwap(d1, d2); // Выводит T = double
}
```

### Шаблоны с несколькими параметрами

```cpp
template <typename T1, typename T2>
void printPair(T1 a, T2 b) {
    std::cout << a << " и " << b << "\n";
}

int main() {
    printPair(10, "Яблоки"); // T1=int, T2=const char*
}
```

## 3. Явная специализация шаблонной функции

Иногда обобщенный алгоритм не подходит для конкретного типа (например, сравнение строк `const char*` через `>` будет сравнивать указатели, а не текст). В таком случае можно создать **явную специализацию** (Explicit Specialization).

```cpp
template <typename T>
T myMax(T a, T b) {
    return (a > b) ? a : b;
}

// Явная специализация для const char*
template <>
const char* myMax<const char*>(const char* a, const char* b) {
    return (std::strcmp(a, b) > 0) ? a : b;
}
```

### Перегрузка шаблонных функций

Шаблоны можно перегружать обычными функциями. При вызове компилятор ищет точное совпадение:

1. Ищет обычную (не шаблонную) функцию.
2. Ищет шаблон, из которого можно сгенерировать точное совпадение.
3. Пытается выполнить стандартные преобразования типов для обычной функции.

## 4. Шаблонные классы

Шаблоны могут применяться к целым классам. Типичный пример — структуры данных (стек, список, вектор).

```cpp
template <typename T>
class Stack {
private:
    T* data;
    int top;
    int capacity;
public:
    Stack(int size);
    ~Stack();
    void push(T item);
    T pop();
};

// Реализация методов вне класса (ОБЯЗАТЕЛЬНО указывать template <typename T>)
template <typename T>
Stack<T>::Stack(int size) {
    data = new T[size];
    top = 0;
    capacity = size;
}

template <typename T>
void Stack<T>::push(T item) {
    if (top < capacity) data[top++] = item;
}
```

**Создание объекта шаблонного класса:**
При создании объекта класса, в отличие от функций, до C++17 мы **обязаны** явно указывать тип в угловых скобках `< >`.

```cpp
Stack<int> intStack(100);
Stack<std::string> strStack(50);
```

## 5. Параметры шаблона, не являющиеся типами (Non-type parameters)

Шаблон может принимать не только типы, но и конкретные значения (константы времени компиляции: `int`, `char`, указатели).

```cpp
template <typename T, int size>
class StaticArray {
private:
    T arr[size]; // Размер известен во время компиляции
public:
    T& operator[](int index) { return arr[index]; }
    int getSize() const { return size; }
};

int main() {
    StaticArray<int, 10> myArr; // Создаст массив из 10 int
    StaticArray<double, 50> myArr2; // Массив из 50 double
    // myArr = myArr2; // ОШИБКА: это абсолютно разные типы!
}
```

## 6. Значения по умолчанию для параметров шаблона

Можно задать типы или значения по умолчанию (как для аргументов функции).

```cpp
template <typename T = int, int size = 100>
class Buffer {
    T data[size];
};

int main() {
    Buffer<> b1;               // T = int, size = 100
    Buffer<double> b2;         // T = double, size = 100
    Buffer<char, 256> b3;      // T = char, size = 256
}
```

## 7. Частичная специализация классов

Для функций доступна только _полная_ специализация. Для классов доступна **частичная специализация** — когда мы фиксируем только часть параметров или накладываем на них условия.

```cpp
template <typename T1, typename T2>
class Pair {
public:
    void info() { std::cout << "Обычный шаблон\n"; }
};

// Частичная специализация: когда оба типа одинаковые
template <typename T>
class Pair<T, T> {
public:
    void info() { std::cout << "Частичная специализация (T, T)\n"; }
};

// Частичная специализация: когда второй тип - int
template <typename T>
class Pair<T, int> {
public:
    void info() { std::cout << "Частичная специализация (T, int)\n"; }
};

// Полная специализация
template <>
class Pair<int, int> {
public:
    void info() { std::cout << "Полная специализация (int, int)\n"; }
};
```

Другой частый пример: частичная специализация для **указателей**:

```cpp
template <typename T> class Storage {};            // Базовый
template <typename T> class Storage<T*> {};        // Для указателей
template <typename T> class Storage<const T*> {};  // Для константных указателей
```

## 8. Зависимые имена и ключевое слово typename

Ключевое слово `typename` используется не только для объявления параметров шаблона. Оно **обязательно**, если вы пытаетесь обратиться к типу, который вложен внутрь шаблонного параметра (зависимое имя).

```cpp
template <typename T>
class Printer {
public:
    void print(T container) {
        // T::iterator зависит от T. Компилятор не знает, это тип или статическая переменная.
        // Добавляем typename, чтобы сказать, что это ТИП.
        typename T::iterator it = container.begin();

        while(it != container.end()) {
            std::cout << *it << " ";
            ++it;
        }
    }
};
```

## 9. Наследование и шаблоны

При наследовании от шаблонного класса нужно понимать: методы базового класса не видны напрямую в производном классе (из-за правил двухфазного поиска имен).

```cpp
template <typename T>
class Base {
protected:
    T value;
public:
    void printValue() { std::cout << value; }
};

template <typename T>
class Derived : public Base<T> {
public:
    void show() {
        // value = 10; // ОШИБКА: компилятор не видит value
        this->value = 10; // ПРАВИЛЬНО
        this->printValue(); // ПРАВИЛЬНО
    }
};
```

_Правило:_ В шаблонном наследнике обращайтесь к членам шаблонного предка через `this->` или `Base<T>::`.

## 10. Вариадические шаблоны (Variadic Templates, C++11)

Шаблоны могут принимать переменное количество параметров. Это делается с помощью "пакета параметров" (parameter pack) `...`.

```cpp
// Базовый случай для остановки рекурсии
void printAll() {
    std::cout << "\n";
}

// Шаблон с переменным числом аргументов
template <typename T, typename... Args>
void printAll(T first, Args... rest) {
    std::cout << first << " ";
    printAll(rest...); // Рекурсивный вызов, распаковка пакета
}

int main() {
    printAll(1, 2.5, "Hello", 'c');
    // Выведет: 1 2.5 Hello c
}
```

## Блок 8: Практические мини-проекты с Шаблонами (Задачи 191–200)

**191.** **Умный указатель `UniquePtr<T>`**: Шаблонный класс, RAII, перегрузка `->` и `*`, удален конструктор копирования, реализована семантика перемещения (Move-семантика).
**192.** **Умный указатель `SharedPtr<T>`**: Шаблонный класс с подсчетом ссылок. Шаблонный Control Block, корректный деструктор, копирование увеличивает счетчик.
**193.** **Кастомный `Vector<T>`**: Полная реализация динамического массива (как std::vector) с аллокацией сырой памяти `operator new[]`, Placement New и ручным вызовом деструкторов для корректной работы с нетривиальными типами.
**194.** **Собственный `List<T>`**: Двусвязный шаблонный список. Итератор списка (класс `ListIterator`, перегрузка `++`, `--`, `*`, `!=`). Поддержка range-based for.
**195.** **Пул объектов (Object Pool)**: `template <typename T, size_t ChunkSize> class ObjectPool`. Предвыделяет память кусками, метод `acquire()` возвращает указатель на память, `release(T*)` возвращает объект в пул.
**196.** **Делегат/Обертка функции (`std::function` своими руками)**: Использование type erasure для сохранения любой функции (указатель на функцию, функтор, лямбда) с заданной сигнатурой `template <typename R, typename... Args> class Function`.
**197.** **Матричная библиотека**: `template <typename T, size_t Rows, size_t Cols> class Matrix`. Перегрузка +, -, \*. Специализация для 1x1, 2x2, 3x3 для оптимизации. Поддержка вывода `<<`.
**198.** **Собственный `Variant` (Tagged Union)**: Использование вариадических шаблонов и объединений (`union`) для хранения одного из заданных типов. Поле метки `type_id`.
**199.** **Сигналы и Слоты (Event System)**: `template <typename... Args> class Signal`. Позволяет подписываться (`connect`) обычным функциям, методам классов (через связывание указателя на объект) и вызывать их (`emit(Args...)`).
**200.** **Контейнер-граф (Graph)**: `template <typename V, typename E> class Graph`. Хранение вершин (данные типа V) и рёбер (вес/данные типа E). Методы добавления, удаления, поиска (DFS/BFS), реализованные шаблонно.

## Рекомендации по отладке шаблонов

1. **Ошибки компиляции шаблонов** (так называемые "портянки") бывают огромными. Всегда ищите корень ошибки в самом верху вывода компилятора, где указано "в месте инстанцирования (instantiated from)".
2. Шаблоны компилируются дважды: сначала парсинг синтаксиса, затем инстанцирование с конкретным типом. Ошибка может появиться только при вызове с определенным типом.
3. Код шаблонных классов и функций (их реализацию) помещайте в заголовочные файлы (`.h` или `.hpp`), так как компилятору нужен доступ к исходному коду для генерации инстанций в точке использования.
