# Перегрузка операторов в C++

---

## 1. Введение в перегрузку операторов

**Перегрузка операторов** — механизм C++, позволяющий переопределить поведение стандартных операторов (`+`, `-`, `*`, `<<`, `[]`, `()` и др.) для пользовательских типов данных (классов и структур).

Идея проста: вы объявляете специальную функцию с именем `operator<символ>`, и компилятор вызывает её каждый раз, когда встречает этот оператор с вашим типом.

```cpp
// Без перегрузки:
Point p3 = p1.add(p2);   // некрасиво

// С перегрузкой:
Point p3 = p1 + p2;       // естественно
```

### Зачем это нужно?

- Делает код **читаемым и интуитивным** (математические типы: вектор, матрица, комплексное число)
- Позволяет работать с пользовательскими типами **так же, как со встроенными**
- Основа для реализации **STL-совместимых** классов

---

## 2. Синтаксис перегрузки

Общая форма:

```cpp
возвращаемый_тип operator<оператор>(список_параметров) {
    // тело функции
}
```

Перегрузка может быть реализована двумя способами:

| Способ                | Синтаксис                                    | Когда использовать                             |
| --------------------- | -------------------------------------------- | ---------------------------------------------- |
| Метод класса          | `T operator+(const T& rhs) const`            | Левый операнд — объект класса                  |
| Дружественная функция | `friend T operator+(const T& a, const T& b)` | Нужен доступ к приватным полям обоих операндов |
| Свободная функция     | `T operator+(const T& a, const T& b)`        | Доступ только через публичный интерфейс        |

### Пример: базовый класс Vector2D

```cpp
#include <iostream>

class Vector2D {
public:
    double x, y;

    Vector2D(double x = 0, double y = 0) : x(x), y(y) {}

    // Перегрузка + как метод класса
    Vector2D operator+(const Vector2D& rhs) const {
        return Vector2D(x + rhs.x, y + rhs.y);
    }

    // Перегрузка - как метод класса
    Vector2D operator-(const Vector2D& rhs) const {
        return Vector2D(x - rhs.x, y - rhs.y);
    }

    // Перегрузка << как дружественная функция
    friend std::ostream& operator<<(std::ostream& os, const Vector2D& v) {
        os << "(" << v.x << ", " << v.y << ")";
        return os;
    }
};

int main() {
    Vector2D a(1, 2), b(3, 4);
    std::cout << a + b << std::endl; // (4, 6)
    std::cout << a - b << std::endl; // (-2, -2)
}
```

## 3. Ограничения перегрузки операторов

### Что НЕЛЬЗЯ делать:

1. **Создавать новые операторы** — нельзя придумать `**` для возведения в степень
2. **Менять приоритет** операторов — `*` всегда выше `+`
3. **Менять арность** — бинарный оператор останется бинарным
4. **Перегружать для встроенных типов** — нельзя изменить поведение `int + int`
5. **Перегружать следующие операторы:**

| Оператор            | Описание                       |
| ------------------- | ------------------------------ |
| `::`                | Разрешение области видимости   |
| `.`                 | Доступ к члену                 |
| `.*`                | Доступ к члену через указатель |
| `?:`                | Тернарный оператор             |
| `sizeof`            | Размер типа                    |
| `typeid`            | Информация о типе              |
| `static_cast` и др. | Операторы приведения типов     |

### Рекомендации (правила хорошего тона):

- Сохраняйте **семантику**: `+` должен складывать, а не вычитать
- Реализуйте парные операторы: если есть `==`, добавьте `!=`
- Если перегружаете `<`, реализуйте все 6 операторов сравнения
- Унарный `+` и `-` не должны изменять объект

## 4. Перегрузка бинарных арифметических операторов

### Операторы +, -, \*, /

```cpp
class Complex {
public:
    double real, imag;

    Complex(double r = 0, double i = 0) : real(r), imag(i) {}

    Complex operator+(const Complex& rhs) const {
        return Complex(real + rhs.real, imag + rhs.imag);
    }

    Complex operator-(const Complex& rhs) const {
        return Complex(real - rhs.real, imag - rhs.imag);
    }

    Complex operator*(const Complex& rhs) const {
        return Complex(
            real * rhs.real - imag * rhs.imag,
            real * rhs.imag + imag * rhs.real
        );
    }

    Complex operator/(const Complex& rhs) const {
        double denom = rhs.real * rhs.real + rhs.imag * rhs.imag;
        return Complex(
            (real * rhs.real + imag * rhs.imag) / denom,
            (imag * rhs.real - real * rhs.imag) / denom
        );
    }

    // Умножение на скаляр (double)
    Complex operator*(double scalar) const {
        return Complex(real * scalar, imag * scalar);
    }

    friend std::ostream& operator<<(std::ostream& os, const Complex& c) {
        os << c.real;
        if (c.imag >= 0) os << "+";
        os << c.imag << "i";
        return os;
    }
};

// Скаляр * Complex (коммутативность)
Complex operator*(double scalar, const Complex& c) {
    return c * scalar;
}
```

### Составные операторы +=, -=, \*=, /=

```cpp
class Complex {
    // ...
    Complex& operator+=(const Complex& rhs) {
        real += rhs.real;
        imag += rhs.imag;
        return *this;   // возвращаем ссылку на себя
    }

    Complex& operator-=(const Complex& rhs) {
        real -= rhs.real;
        imag -= rhs.imag;
        return *this;
    }
};
```

> **Важно:** `+=` и подобные операторы **возвращают ссылку** `T&`, чтобы поддерживать цепочки присваивания и эффективность.

## 5. Перегрузка операторов сравнения

```cpp
class Fraction {
    int num, den;
public:
    Fraction(int n, int d) : num(n), den(d) {}

    bool operator==(const Fraction& rhs) const {
        return num * rhs.den == rhs.num * den;
    }

    bool operator!=(const Fraction& rhs) const {
        return !(*this == rhs);   // используем уже перегруженный ==
    }

    bool operator<(const Fraction& rhs) const {
        return num * rhs.den < rhs.num * den;
    }

    bool operator>(const Fraction& rhs) const {
        return rhs < *this;
    }

    bool operator<=(const Fraction& rhs) const {
        return !(rhs < *this);
    }

    bool operator>=(const Fraction& rhs) const {
        return !(*this < rhs);
    }
};
```

> **Совет от Шилдта:** определите `==` и `<`, остальные выразите через них.

## 6. Перегрузка унарных операторов

### Унарный минус, плюс, логическое отрицание

```cpp
class Vector2D {
public:
    double x, y;
    Vector2D(double x, double y) : x(x), y(y) {}

    // Унарный минус (инвертирует направление)
    Vector2D operator-() const {
        return Vector2D(-x, -y);
    }

    // Унарный плюс (возвращает копию)
    Vector2D operator+() const {
        return *this;
    }

    // Логическое отрицание (нулевой вектор?)
    bool operator!() const {
        return (x == 0 && y == 0);
    }
};
```

### Операторы инкремента и декремента (++ и --)

```cpp
class Counter {
    int value;
public:
    Counter(int v = 0) : value(v) {}

    // Префиксный ++: ++obj
    Counter& operator++() {
        ++value;
        return *this;
    }

    // Постфиксный ++: obj++  (параметр int — фиктивный, нужен для различения)
    Counter operator++(int) {
        Counter temp = *this;   // сохраняем старое значение
        ++value;
        return temp;            // возвращаем копию старого значения
    }

    // Префиксный --
    Counter& operator--() {
        --value;
        return *this;
    }

    // Постфиксный --
    Counter operator--(int) {
        Counter temp = *this;
        --value;
        return temp;
    }

    int get() const { return value; }
};
```

> **Ключевое различие:**
>
> - Префиксный возвращает **ссылку** (`T&`) на изменённый объект
> - Постфиксный возвращает **копию** (`T`) старого значения
> - Постфиксный принимает фиктивный параметр `int`

## 7. Перегрузка операторов ввода/вывода (>> и <<)

Операторы `<<` и `>>` **ВСЕГДА** реализуются как **дружественные или свободные функции**, т.к. левый операнд — `std::ostream` / `std::istream`, а не наш класс.

```cpp
class Point {
    double x, y;
public:
    Point(double x = 0, double y = 0) : x(x), y(y) {}

    // Оператор вывода
    friend std::ostream& operator<<(std::ostream& os, const Point& p) {
        os << "Point(" << p.x << ", " << p.y << ")";
        return os;   // возвращаем поток для цепочки: cout << p1 << p2;
    }

    // Оператор ввода
    friend std::istream& operator>>(std::istream& is, Point& p) {
        is >> p.x >> p.y;
        return is;
    }
};

int main() {
    Point p;
    std::cin >> p;                          // вводим точку
    std::cout << p << std::endl;            // выводим точку
    std::cout << Point(1,2) << Point(3,4);  // цепочка вывода
}
```

## 8. Перегрузка оператора индексирования []

```cpp
class SafeArray {
    int* data;
    int size;
public:
    SafeArray(int sz) : size(sz), data(new int[sz]()) {}
    ~SafeArray() { delete[] data; }

    // Версия для чтения и записи
    int& operator[](int index) {
        if (index < 0 || index >= size)
            throw std::out_of_range("Индекс за пределами массива");
        return data[index];
    }

    // Версия только для чтения (константный объект)
    const int& operator[](int index) const {
        if (index < 0 || index >= size)
            throw std::out_of_range("Индекс за пределами массива");
        return data[index];
    }
};
```

## 9. Перегрузка оператора вызова функции ()

Объект, у которого перегружен `()`, называется **функтором** (функциональный объект).

```cpp
class Multiplier {
    int factor;
public:
    Multiplier(int f) : factor(f) {}

    // Оператор ()
    int operator()(int x) const {
        return x * factor;
    }
};

int main() {
    Multiplier triple(3);
    std::cout << triple(5);  // выводит 15
    std::cout << triple(7);  // выводит 21
}
```

Функторы широко применяются в алгоритмах STL:

```cpp
#include <algorithm>
#include <vector>

class IsEven {
public:
    bool operator()(int n) const { return n % 2 == 0; }
};

int main() {
    std::vector<int> v = {1, 2, 3, 4, 5, 6};
    auto it = std::find_if(v.begin(), v.end(), IsEven());
}
```

## 11. Перегрузка операторов преобразования типов

```cpp
class Fraction {
    int num, den;
public:
    Fraction(int n, int d) : num(n), den(d) {}

    // Преобразование в double
    operator double() const {
        return static_cast<double>(num) / den;
    }

    // Преобразование в bool (явное)
    explicit operator bool() const {
        return num != 0;
    }
};

int main() {
    Fraction f(3, 4);
    double d = f;       // неявное: вызывает operator double()
    if (f) { ... }      // явное: вызывает operator bool()
}
```

> **Важно:** используйте `explicit`, чтобы избежать неожиданных неявных преобразований.

## 12. Перегрузка оператора присваивания =

```cpp
class Matrix {
    int rows, cols;
    double* data;
public:
    Matrix(int r, int c) : rows(r), cols(c), data(new double[r * c]()) {}

    // Конструктор копирования
    Matrix(const Matrix& other)
        : rows(other.rows), cols(other.cols),
          data(new double[other.rows * other.cols])
    {
        std::copy(other.data, other.data + rows * cols, data);
    }

    // Оператор присваивания (идиома copy-and-swap)
    Matrix& operator=(Matrix other) {   // параметр по значению!
        std::swap(rows, other.rows);
        std::swap(cols, other.cols);
        std::swap(data, other.data);
        return *this;
    }

    ~Matrix() { delete[] data; }
};
```

> **Правило трёх (Rule of Three):** если вы определяете деструктор, конструктор копирования или оператор присваивания — определяйте все три.

> **Правило пяти (Rule of Five):** в C++11 добавляются перемещающий конструктор и перемещающее присваивание.

## 13. Перегрузка оператора new и delete

```cpp
#include <cstdlib>
#include <iostream>

class Tracked {
public:
    static int count;

    void* operator new(std::size_t size) {
        ++count;
        std::cout << "Выделение памяти #" << count << std::endl;
        return std::malloc(size);
    }

    void operator delete(void* ptr) {
        --count;
        std::cout << "Освобождение памяти. Живых объектов: " << count << std::endl;
        std::free(ptr);
    }
};

int Tracked::count = 0;
```

## 14. Перегрузка операторов для шаблонных классов

```cpp
template<typename T>
class Pair {
    T first, second;
public:
    Pair(T a, T b) : first(a), second(b) {}

    Pair operator+(const Pair& rhs) const {
        return Pair(first + rhs.first, second + rhs.second);
    }

    bool operator==(const Pair& rhs) const {
        return first == rhs.first && second == rhs.second;
    }

    template<typename U>
    friend std::ostream& operator<<(std::ostream& os, const Pair<U>& p);
};

template<typename T>
std::ostream& operator<<(std::ostream& os, const Pair<T>& p) {
    return os << "{" << p.first << ", " << p.second << "}";
}
```

## 15. Метод vs. Дружественная функция: когда что использовать

| Критерий                        | Метод класса                | Дружественная функция |
| ------------------------------- | --------------------------- | --------------------- |
| Доступ к приватным полям        | Да                          | Да                    |
| Левый операнд                   | Должен быть объектом класса | Любой тип             |
| Операторы `=`, `[]`, `()`, `->` | **Обязательно метод**       | Нельзя                |
| Операторы `<<`, `>>`            | Нельзя                      | **Обязательно**       |
| Симметричные бинарные операторы | Неудобно                    | Удобно                |

## 16. Полный пример: класс String

```cpp
#include <iostream>
#include <cstring>
#include <stdexcept>

class MyString {
    char* data;
    size_t len;
public:
    MyString(const char* s = "") : len(strlen(s)), data(new char[len + 1]) {
        strcpy(data, s);
    }

    MyString(const MyString& other) : len(other.len), data(new char[len + 1]) {
        strcpy(data, other.data);
    }

    MyString& operator=(MyString other) {
        std::swap(data, other.data);
        std::swap(len, other.len);
        return *this;
    }

    ~MyString() { delete[] data; }

    // Конкатенация
    MyString operator+(const MyString& rhs) const {
        MyString result;
        delete[] result.data;
        result.len = len + rhs.len;
        result.data = new char[result.len + 1];
        strcpy(result.data, data);
        strcat(result.data, rhs.data);
        return result;
    }

    MyString& operator+=(const MyString& rhs) {
        *this = *this + rhs;
        return *this;
    }

    // Сравнение
    bool operator==(const MyString& rhs) const { return strcmp(data, rhs.data) == 0; }
    bool operator<(const MyString& rhs) const  { return strcmp(data, rhs.data) < 0; }
    bool operator>(const MyString& rhs) const  { return rhs < *this; }

    // Индексирование
    char& operator[](size_t i) {
        if (i >= len) throw std::out_of_range("Index out of range");
        return data[i];
    }

    const char& operator[](size_t i) const {
        if (i >= len) throw std::out_of_range("Index out of range");
        return data[i];
    }

    // Ввод/Вывод
    friend std::ostream& operator<<(std::ostream& os, const MyString& s) {
        return os << s.data;
    }

    friend std::istream& operator>>(std::istream& is, MyString& s) {
        char buf[1024];
        is >> buf;
        s = MyString(buf);
        return is;
    }

    size_t size() const { return len; }
};
```

## 17. C++20: оператор трёхстороннего сравнения <=>

В C++20 введён оператор `<=>` (spaceship operator), позволяющий автоматически сгенерировать все 6 операторов сравнения:

```cpp
#include <compare>

class Version {
    int major, minor, patch;
public:
    Version(int maj, int min, int pat)
        : major(maj), minor(min), patch(pat) {}

    // Определяем один оператор — компилятор генерирует все 6
    auto operator<=>(const Version& rhs) const = default;

    // Можно и вручную:
    // std::strong_ordering operator<=>(const Version& rhs) const {
    //     if (major != rhs.major) return major <=> rhs.major;
    //     if (minor != rhs.minor) return minor <=> rhs.minor;
    //     return patch <=> rhs.patch;
    // }
};

int main() {
    Version v1(1, 2, 3), v2(1, 3, 0);
    if (v1 < v2) std::cout << "v1 старше";
}
```

## 18. Итоговая таблица: виды операторов и способы перегрузки

| Оператор             | Тип       | Способ           | Возвращает              |
| -------------------- | --------- | ---------------- | ----------------------- |
| `+`, `-`, `*`, `/`   | Бинарный  | Метод или friend | Новый объект `T`        |
| `+=`, `-=`, `*=`     | Бинарный  | Метод            | Ссылку `T&`             |
| `==`, `!=`, `<`, `>` | Бинарный  | Метод или friend | `bool`                  |
| `<<`, `>>`           | Бинарный  | Только friend    | `ostream&` / `istream&` |
| `=`                  | Бинарный  | Только метод     | Ссылку `T&`             |
| `[]`                 | Бинарный  | Только метод     | Ссылку `T&`             |
| `()`                 | Н-арный   | Только метод     | Любой тип               |
| `->`                 | Унарный   | Только метод     | Указатель               |
| `++` (префикс)       | Унарный   | Метод            | Ссылку `T&`             |
| `++` (постфикс)      | Унарный   | Метод            | Копию `T`               |
| `-` (унарный)        | Унарный   | Метод или friend | Новый объект `T`        |
| `type()`             | Конверсия | Только метод     | Указанный тип           |
